/ 项目简介 概述

将 S3 兼容对象存储零配置地嵌入 Spring Boot 应用的启动器， 一行依赖即可操作任意 S3 服务。

基于 Spring Boot 2.6.11 与 AWS SDK for Java v1.12.709 构建， 提供开箱即用的 AmazonS3Template。 仅需添加 Maven 依赖并配置 winter-aws.* 属性 — 即可开始上传、下载、管理对象和存储桶。

全面封装 S3 协议操作：存储桶管理、对象 CRUD、预签名 URL、 分片上传、元数据管理、版本控制、生命周期和事件通知配置。 兼容 MinIO、AWS S3、阿里云 OSS、腾讯云 COS、华为云 OBS、七牛云 Kodo。

为什么选择 Winter MinIO
  • 传统方式通过 AWS SDK 操作 S3 需要编写大量样板代码：客户端构造、连接池配置、异常处理等。
  • 本 Starter 通过 Auto-Configuration 自动化这一切，一行配置即可获得完全配置的 AmazonS3 客户端。
  • Template 模式封装 80+ 常用操作，覆盖日常 95% 以上的 S3 使用场景。
  • 内置三种存储桶策略模板（只读/只写/读写），无需手动编写复杂的 JSON Policy。
  • 支持 Path-Style 与 Virtual-Hosted 双模式，一套代码切换私有部署和公有云。

/ 核心能力 特性

01 存储桶全生命周期管理 创建、删除、检查存储桶是否存在。内置三种策略模板（READ_ONLY / WRITE_ONLY / READ_WRITE），自动生成标准 S3 Policy JSON，无需手写策略文档。 核心
02 对象 CRUD 与批量操作 上传（InputStream / MultipartFile / byte[]）、下载（S3Object / InputStream / 本地文件）、删除、复制、重命名、批量删除。支持前缀列举和分页查询。 核心
03 预签名 URL 生成 生成带过期时间的 GET / PUT / DELETE 预签名 URL。支持自定义过期时间（秒/分/时）、自定义 HTTP 方法、自定义域名网关 URL，适合前端直传和临时分享。 网络
04 分片上传与大文件支持 初始化分片上传、逐个上传分片、列出已上传分片、完成或中止分片。适用于大文件场景，避免 OOM。列出所有进行中的分片任务。 核心
05 元数据管理与版本控制 获取/设置对象的自定义元数据（Content-Type、用户自定义键值对）。启用/禁用存储桶版本控制，列出所有版本历史，追踪文件变更。 管理
06 生命周期与事件通知 配置存储桶生命周期规则（过期天数、迁移到 Glacier）。设置存储桶事件通知（对象创建/删除事件），支持 SNS、SQS、Lambda 目标。 管理

/ 架构设计 架构

自动装配流水线

Spring Boot 启动时，Auto-Configuration 按以下顺序依次创建并装配各组件。 标记红色的为可替换 Bean —— 开发者可通过 @Bean 提供自定义实现覆盖默认。

Properties → ClientConfig → AmazonS3 → S3Template → CommandRunner

各组件职责

AmazonS3Properties @ConfigurationProperties(prefix="winter-aws")，绑定 accessKey、secretKey、endpoint、bucket、region、pathStyleAccess、customDomain
ClientConfiguration 通过 Properties 驱动的传输参数：maxConnections、socketTimeout、connectionTimeout、maxErrorRetry。均为可配置项，支持 application.yml 自定义
AmazonS3 Client 通过 AmazonS3ClientBuilder 构建，支持 path-style 与 virtual-hosted 双模式。执行实际的 S3 REST API 调用
AmazonS3Template 核心门面类，封装 80+ 操作方法。基于 @ConditionalOnMissingBean，可被用户自定义 Bean 替换
BucketInitCommandLineRunner 应用启动后检查默认 Bucket 是否存在，不存在则自动创建（配置 createBucketOnStartup=true）。日志输出 bucket 信息便于验证

自动装配时序

每一步判断条件是否满足，条件不满足则跳过对应的 Bean 创建：

@ConditionalOnProperty → @EnableConfigurationProperties → @ConditionalOnMissingBean → @ConditionalOnBean → @Bean init/destroy

组件依赖树

Bean 加载顺序

Starter 按照严格的依赖顺序注册 Bean， 后置 Bean 依赖前置 Bean 的存在：

Properties → ClientConfig → AmazonS3 → S3Template → BucketInit

文件上传生命周期

一次完整的文件上传请求经过以下层级：

01 Controller 接收 MultipartFile
02 AmazonS3Template.putObject() 调用
03 PutObjectRequest 构建（含元数据）
04 AmazonS3.putObject() SDK 调用
05 返回 PutObjectResult（ETag）
06 生成访问 URL / 预签名 URL

文件下载生命周期

流式下载请求经过以下层级，避免将大文件加载到内存：

01 Controller 接收下载请求（key 参数）
02 AmazonS3Template.getObject() 调用
03 S3Object 返回，包含 ObjectMetadata + 内容流
04 ResponseEntity<StreamingResponseBody> 包装流
05 StreamingResponseBody 逐块写入 OutputStream
06 Stream 自动关闭，资源释放

/ 使用指南 工作流

01
添加 Maven 依赖
在 pom.xml 中加入 dependency。 已发布至 Maven Central，无需配置额外仓库。
<!-- pom.xml --> <dependency> <groupId>io.github.hahaha-zsq</groupId> <artifactId>winter-minio-spring-boot-starter</artifactId> <version>0.0.3</version> </dependency>
02
配置 application.yml
四项必填参数：access-key、 secret-key、 endpoint、 bucket。 生产环境推荐使用 ${ENV_VAR} 环境变量注入密钥。
# application.yml — 完整配置参考 winter-aws: access-key: ${AWS_ACCESS_KEY} secret-key: ${AWS_SECRET_KEY} endpoint: http://localhost:9000 bucket: my-bucket # region: AP_EAST_1 # protocol: http # path-style-access: true # max-connections: 500 # socket-timeout: 20000 # connection-timeout: 10000 # max-error-retry: 2 # custom-domain: https://cdn.example.com # enabled: true
03
注入 AmazonS3Template
在任意 @Service 或 @RestController 中 @Autowired 注入 AmazonS3Template。 Starter 自动创建并管理其生命周期。
// 构造器注入示例 @Service public class FileService { private final AmazonS3Template s3; public FileService(AmazonS3Template s3) { this.s3 = s3; } public String upload(MultipartFile file) { String key = "uploads/" + file.getOriginalFilename(); s3.putObject(key, file, null); return s3.getGatewayUrl(key); } }
04
执行对象操作
调用核心 API 完成文件的上传、下载、删除、复制。 支持 MultipartFile、InputStream、byte[] 多种输入方式。
// 上传 + 下载 + 删除一览 // 上传 (InputStream) s3.putObject(key, inputStream); // 上传 (MultipartFile + 元数据) s3.putObject(key, multipartFile, metadata); // 上传 (byte[] + Content-Type) s3.putObject(key, bytes, "image/png"); // 下载 (返回 S3Object) S3Object obj = s3.getObject(key); // 下载 (返回 InputStream) InputStream in = s3.getObjectInputStream(key); // 范围读取 (断点续传) InputStream range = s3.getObjectInputStream(key, start, end); // 删除 s3.removeObject(key); // 批量删除 s3.removeObjects(bucket, Arrays.asList(key1, key2, key3));
05
生成访问链接
使用 getGatewayUrl 获取公开访问链接， 或预签名 URL 安全分享文件。支持自定义过期时间与 HTTP 方法。
// 公开访问（需 bucket 策略允许） String url = s3.getGatewayUrl(key); // 预签名 GET URL（15 分钟有效） String presigned = s3.getObjectUrl(key, 15, TimeUnit.MINUTES); // 预签名 PUT URL（10 分钟有效，前端直传） String putUrl = s3.getPresignedObjectPutUrl(bucket, key, 10); // 自定义方法预签名 String custom = s3.getObjectUrl(bucket, key, 1, TimeUnit.HOURS, HttpMethod.DELETE);
06
配置生命周期策略
设置过期规则自动清理临时文件，或配置版本控制追踪文件变更。 事件通知可对接 SNS / SQS / Lambda。
// 30 天后自动删除 BucketLifecycleConfiguration config = new BucketLifecycleConfiguration(); List<Rule> rules = new ArrayList<>(); rules.add(new Rule() .withId("temp-expiry") .withExpirationInDays(30) .withStatus(BucketLifecycleConfiguration.ENABLED)); config.setRules(rules); s3.setBucketLifecycleConfiguration(config); // 启用版本控制 s3.enableBucketVersioning(); // 列出所有版本 s3.listVersions(prefix);
// 完整示例：文件上传 REST 控制器 @RestController @RequiredArgsConstructor public class FileController { private final AmazonS3Template s3; @PostMapping("/upload") public Map<String, Object> upload(@RequestParam MultipartFile file) { String key = "uploads/" + System.currentTimeMillis() + "_" + file.getOriginalFilename(); PutObjectResult result = s3.putObject(key, file, null); return Map.of( "etag", result.getETag(), "url", s3.getGatewayUrl(key) ); } @GetMapping("/download/{key}") public ResponseEntity<StreamingResponseBody> download(@PathVariable String key) { S3Object obj = s3.getObject(key); StreamingResponseBody body = os -> { try (InputStream in = obj.getObjectContent()) { int len; byte[] buf = new byte[8192]; while ((len = in.read(buf)) != -1) os.write(buf, 0, len); } }; return ResponseEntity.ok() .header("Content-Type", obj.getObjectMetadata().getContentType()) .body(body); } }

/ 常见问题 FAQ

Q: 这个 Starter 是否支持 AWS S3 之外的存储服务？

A: 支持。所有兼容 S3 协议的对象存储服务均可使用，包括 MinIO、阿里云 OSS、腾讯云 COS、 华为云 OBS、七牛云 Kodo、DigitalOcean Spaces 等。只需修改 endpoint 和 path-style-access 配置即可无缝切换。

Q: 为什么 AWS SDK v1 而不是 v2？

A: 项目基于 Spring Boot 2.6.11（Java 8），SDK v1 提供更稳定的兼容性。 如需使用 SDK v2，可参考 AmazonS3Template 的封装思路自行迁移。 SDK v2 使用 S3Client（异步为 S3AsyncClient）， 需注意 Spring Boot 3.x 以上才提供 Jakarta 命名空间支持。

Q: 如何配置多个 Bucket？

A: 当前版本配置只支持一个默认 Bucket。多 Bucket 场景可通过以下方式实现： 1）所有需要指定 Bucket 的方法都提供了带 bucketName 参数的重载版本； 2）创建多个 AmazonS3Client Bean 并各自封装 Template； 3）在业务代码中维护 Bucket 与 Template 的映射 Map。推荐方案二，后续版本将考虑内置多 Bucket 支持。

Q: 生产环境推荐使用 HTTPS 吗？如何配置？

A: 强烈推荐。在 application.yml 中将 winter-aws.protocol 设为 https 即可（不区分大小写）， Starter 自动选择 Protocol.HTTPS。 同时建议使用 custom-domain 配合 CDN 的 HTTPS 能力， 在网关层实现加密传输。

Q: 文件上传报 Connection timed out 怎么办？

A: 常见原因及解决方案：1）确认 endpoint 地址正确且网络可达； 2）防火墙/安全组是否放行了对应端口；3）MinIO 默认使用 9000 端口，确认服务已启动； 4）在 application.yml 中调大 winter-aws.connection-timeout（默认 10000ms）； 5）检查 accessKey / secretKey 是否正确，认证失败也会表现为连接异常。

Q: 如何调整 maxConnections 等连接参数？

A: 在 application.yml 中配置即可： winter-aws.max-connections（默认 500）、 winter-aws.socket-timeout（默认 20000ms）、 winter-aws.connection-timeout（默认 10000ms）、 winter-aws.max-error-retry（默认 2）。 所有参数已暴露到 Properties，无需修改源码。

Q: 大文件上传如何避免 OOM？

A: 推荐使用分片上传（Multipart Upload）处理大文件：1）调用 initiateMultipartUpload 获取 UploadId； 2）将文件切分为 5MB~5GB 的分片逐个调用 uploadPart； 3）所有分片上传成功后调用 completeMultipartUpload 合并。 分片上传避免了将整个文件加载到内存，且支持断点续传。

Q: 如何调试 S3 请求？

A: AWS SDK v1 支持通过系统属性或 ClientConfiguration 开启请求/响应日志： 1）添加 log4j.logger.com.amazonaws.request=DEBUG； 2）在 ClientConfiguration 上设置 setRequestMetricCollector； 3）捕获 AmazonServiceException 中的 getErrorCode() 和 getErrorMessage() 获得详细的错误信息。

Q: 如何将文件列表分页？

A: 使用 getAllObjectsByPrefix(prefix, delimiter, maxKeys) 方法，返回 ObjectListing 对象。 通过 objectListing.getNextMarker() 获取下一页标记，传入下一次请求的 marker 参数。 S3 默认每次最多返回 1000 条记录，可通过 maxKeys 调整。

Q: 是否支持 Spring Boot 3.x？

A: 当前版本基于 Spring Boot 2.6.11 和 Java 8，使用 javax 命名空间，不直接兼容 Spring Boot 3.x（jakarta 命名空间）。如需迁移至 Spring Boot 3.x，需要：1）将 javax.* 替换为 jakarta.*； 2）升级 AWS SDK 至 v2；3）验证 spring.factories 在新版本中的兼容性。 计划在未来版本提供 Spring Boot 3.x 支持。
